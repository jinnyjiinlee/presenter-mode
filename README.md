# 발표자 모드 (Presenter Mode)

> HTML 슬라이드에 파워포인트급 **"발표자 보기"** 를 더해요.
> **파일 하나 · 설치 없음 · 인터넷 없음.**

![발표자 창](docs/presenter-view.png)

발표자 창에서는 현재/다음 슬라이드·강사 노트·타이머·전체 필름스트립이 보이고, 강의장 화면(빔프로젝터)에는 슬라이드만 깨끗하게 나가요. 실제 기업·기관 강의(7시간 풀데이 포함)에서 쓰고 있는 도구예요.

**목차** · [① 3분 체험](#-1-일단-3분-체험) · [② 내 슬라이드에 붙이기](#-2-내-슬라이드에-붙이기--방법은-2가지) · [③ 기능](#-3-기능-한눈에) · [④ 단축키](#-4-단축키) · [⑤ 강의장 세팅](#-5-강의장-세팅-체크리스트) · [⑥ 작동 원리](#-6-어떻게-동작하나요) · [FAQ](#faq)

## ▶ 1. 일단 3분 체험

1. 초록색 `Code` 버튼 → `Download ZIP` → 압축 풀기
2. `index.html` 더블클릭 (크롬으로 열려요)
3. 키보드 **`P`** — 발표자 창 등장!

데모 슬라이드 6장이 곧 튜토리얼이에요. 방향키로 넘기면서 노트 강조·`B` 블랙아웃·숫자 점프를 직접 눌러보세요.

## ▶ 2. 내 슬라이드에 붙이기 — 방법은 2가지

|  | **방법 1 · AI에게 시키기** | **방법 2 · 직접 붙이기** |
|---|---|---|
| 누구에게 | 비개발자, 바쁜 분 ⭐ 추천 | HTML이 편한 분 |
| 걸리는 시간 | 약 5분 | 10~20분 |
| 필요한 것 | 클로드코드 등 AI 코딩 도구 | 텍스트 에디터 |

### 방법 1 · AI에게 시키기 ⭐

클로드코드(또는 다른 AI 코딩 도구)에 아래 프롬프트를 그대로 붙여넣으세요. **대괄호 `[ ]` 두 곳만** 여러분 것으로 바꾸면 돼요.

```text
아래 저장소의 "발표자 모드"를 내 슬라이드 파일에 붙여줘.

저장소: https://github.com/jinnyjiinlee/presenter-mode
내 슬라이드 파일: [내 슬라이드 HTML 파일 경로. 예: ~/Desktop/발표자료.html]

작업 순서:
1. 저장소의 index.html에서 발표자 모드 엔진을 통째로 가져와
   (CSS의 "발표자 모드 (Presenter View)" 섹션부터 끝까지 + <script> 블록 전체)
2. 내 슬라이드 파일 구조를 먼저 읽고, 슬라이드 한 장이 <section class="slide">
   하나가 되도록 맞춰줘 (첫 장만 class="slide active").
   내 슬라이드가 1600×900 기준이 아니면 .deck-inner/.slide 크기와
   fit() 함수의 계산을 내 슬라이드 크기에 맞게 바꿔줘.
3. NOTES 배열을 내 슬라이드 장수만큼 만들어줘. 내용은 일단 전부 빈 문자열로
   두되, 각 줄에 /*번호 슬라이드제목*/ 주석을 달아서 내가 나중에 채우기 쉽게 해줘.
4. DECK_ID를 '[내 덱 이름. 예: sales-2026]'으로 바꿔줘.
5. 다 되면 크롬으로 열어서 확인해줘: P 키로 발표자 창이 뜨는지,
   방향키로 넘길 때 두 창이 같이 움직이는지, B 키 블랙아웃이 되는지.

주의: 내 원본 슬라이드의 디자인과 내용은 절대 바꾸지 마.
```

끝나면 크롬에서 열고 `P`를 눌러 확인하세요. 노트는 나중에 AI에게 이렇게 시키면 돼요:

```text
3번 슬라이드 노트에 "여기서 질문 받기"라고 넣어줘.
```

### 방법 2 · 직접 붙이기

`index.html`이 곧 템플릿이에요. **세 군데만** 바꾸면 돼요.

**① 슬라이드 교체** — 파일 안 `▼▼▼ 여기부터 교체` 주석 사이의 `<section class="slide">` 들을 여러분 슬라이드로 바꿔요.

```html
<section class="slide">   <!-- 슬라이드 1장 = section.slide 하나 -->
  <!-- 1600×900 기준으로 디자인하면 자동으로 화면에 맞게 스케일돼요 -->
</section>
```

- 첫 슬라이드에만 `class="slide active"` — 내부 디자인은 완전 자유

**② 노트 채우기** — `<script>` 안 `NOTES` 배열에 슬라이드 순서대로 멘트를 적어요.

```js
const NOTES = [
/*01*/ `오프닝 멘트. ⏰ 10:30 컷라인 — 시간 큐는 주황으로 강조돼요.`,
/*02*/ ``,   // 노트 없는 장은 빈 문자열
/*03*/ `⚠️ 이 시연은 와이파이 필요. 백업: backup.png`,
];
```

**③ 덱 이름 정하기** — `DECK_ID`를 덱마다 다르게. (같은 브라우저에서 여러 덱을 써도 슬라이드 위치·타이머가 안 섞이는 열쇠예요)

```js
const DECK_ID='my-lecture-2026';
```

## ▶ 3. 기능 한눈에

**발표자 창 (나만 보는 화면)**

- 현재/다음 슬라이드 — 비율은 드래그로 조절 (기억됨)
- 강사 노트 — `⏰` 시간 큐는 주황, `⚠️` 주의는 빨강, `백업` 줄은 파랑 자동 강조 + 다음 노트 미리보기
- 필름스트립 — 전체 썸네일 클릭 이동, 노트 있는 장 ✎ 배지
- 타이머 — 목표 시간 설정 시 **남은 시간** 표시 (5분 전 알림·초과 시 빨간 깜빡임), 창을 닫아도 이어짐
- 현재 시각 · 슬라이드 체류 시간 · 강의장 화면 연결 상태(●/○)
- 기본은 밝은 배경 — 어두운 강의장에선 🌙 버튼으로 다크 전환 (기억됨)

**강의장 화면 (청중이 보는 화면)**

- `B`/`W` 블랙아웃 — 화면만 검게/희게 (쉬는 시간·집중 유도), 슬라이드 넘기면 자동 해제
- 조작 없으면 4초 뒤 커서·힌트 자동 숨김 — 프로젝터엔 슬라이드만
- 발표 중 도움말·점프창이 청중에게 노출되지 않게 자동 차단
- Wake Lock — 발표 중 화면 꺼짐 방지

**진행**

- 프레젠터 리모컨(클리커) 호환 — Space·Enter·PgUp/PgDn
- 숫자 + Enter로 슬라이드 점프 (Q&A 때 "아까 그 화면" 소환)
- 두 창 자동 동기화 — 어긋나면 2초 안에 자동 복구

## ▶ 4. 단축키

발표자 창 하단 **[? 단축키]** 버튼(또는 `?`)을 누르면 화면에서 바로 볼 수 있어요.

| 키 | 동작 |
|---|---|
| `←` `→` · `Space` · `Enter` · 리모컨 | 슬라이드 이동 |
| 숫자 + `Enter` | 해당 번호로 점프 |
| `Home` / `End` | 처음 / 마지막 |
| `B` / `W` | 강의장 화면 검게/희게 가리기 |
| `F` 또는 `F5` | 전체화면 |
| `P` · `⌘K`(맥) · `Ctrl+K`(윈도우) | 발표자 창 열기 |
| `V` | 발표자 창 보기 전환 |
| `T` / `R` | 타이머 시작·정지 / 리셋(두 번) |
| `?` 또는 `H` | 단축키 도움말 |

## ▶ 5. 강의장 세팅 체크리스트

1. 빔프로젝터 연결 → 디스플레이 **"확장"** 모드 (미러링 끄기)
   - 맥: `⌘+Fn+F1` 토글 · 윈도우: `Win+P`
2. `index.html`(내 덱) 열고 `P` → 발표자 창 등장
3. 슬라이드 창을 프로젝터 화면으로 드래그 → `F` 전체화면
4. 발표자 창 하단 **"● 강의장 화면 연결됨"** 초록불 확인 — 끝!

시연할 때 미러링으로 전환했다가(`⌘+Fn+F1`) 다시 확장으로 돌아와도 슬라이드 위치는 안 흐트러져요.

## ▶ 6. 어떻게 동작하나요

- 같은 HTML 파일을 **창 2개**로 열어요. URL 해시가 `#presenter`면 발표자 UI, 아니면 슬라이드를 렌더링해요.
- 두 창은 `localStorage` + `BroadcastChannel`로 실시간 동기화 — 전부 브라우저 안에서만 일어나서 **인터넷이 필요 없어요.**
- 발표자 창이 2초마다 ping, 슬라이드 창이 pong — 연결 상태 표시와 어긋남 자동 복구의 원리예요.
- 미리보기·썸네일은 실제 슬라이드 DOM을 `cloneNode` + `transform: scale()`로 축소한 것 — 이미지 캡처가 아니라 항상 원본과 똑같아요.
- 타이머·테마·레이아웃 설정은 `localStorage`에 저장돼 창을 닫아도 유지돼요.

**요구 사항**: 크롬/엣지 권장 · 맥·윈도우 지원 · 서버·빌드·설치 불필요 (`file://`로 그냥 열면 돼요)

## FAQ

**Q. 발표자 창을 실수로 닫았어요.**
`P`로 다시 열면 현재 슬라이드·타이머까지 그대로 따라와요.

**Q. 팝업이 차단됐대요.**
주소창 오른쪽 팝업 아이콘에서 "항상 허용" 후, 화면 안내의 [허용했어요 — 다시 열기] 버튼을 누르면 돼요.

**Q. PDF로도 뽑고 싶어요.**
크롬 인쇄(`⌘P`)에서 PDF 저장 — 발표자 UI는 인쇄에서 자동 제외돼요.

**Q. 두 대의 컴퓨터에서 쓸 수 있나요?**
아니요 — 한 컴퓨터의 화면 2개(노트북+프로젝터) 구조예요. 동기화가 브라우저 안에서만 일어나기 때문이에요.

**Q. 파워포인트/키노트 파일에도 붙일 수 있나요?**
아니요, HTML 슬라이드 전용이에요. (요즘은 AI에게 "PPT 내용을 HTML 슬라이드로 만들어줘"라고 한 뒤 붙이는 방법도 있어요)
