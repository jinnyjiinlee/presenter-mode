# 발표자 모드 (Presenter Mode)

**HTML 파일 하나로 끝내는 발표자 모드.** 설치도, 로그인도, 인터넷도 필요 없어요.

파워포인트/키노트의 "발표자 보기"를 순수 HTML/CSS/JS로 구현했어요. 발표자 창에서 현재/다음 슬라이드, 강사 노트, 타이머, 전체 필름스트립을 보면서 — 강의장 화면(빔프로젝터)에는 슬라이드만 깨끗하게 나갑니다.

DAYFOCUS LAB에서 실제 기업·기관 강의(7시간 풀데이 포함)에 쓰고 있는 도구입니다.

![발표자 창](docs/presenter-view.png)

## 왜 만들었나

HTML로 슬라이드를 만들면(요즘 AI로 많이들 만들죠) 예쁘고 가벼운데, 딱 하나가 아쉬워요 — **발표자 노트가 없다는 것.** 이 프로젝트는 HTML 덱에 그 마지막 조각을 채워줍니다.

- **파일 하나가 전부** — `index.html` 안에 슬라이드 + 발표자 모드 엔진이 다 들어있어요
- **오프라인 완전 동작** — 강의장 와이파이가 죽어도 아무 영향 없어요 (모든 동기화는 브라우저 안에서만)
- **비개발자 강사도 OK** — 더블클릭 → `P` 한 번이면 끝

## 빠른 시작 (30초)

1. 이 저장소를 다운로드해요 (초록색 `Code` 버튼 → `Download ZIP`)
2. `index.html`을 더블클릭해서 크롬으로 열어요
3. 키보드 **`P`** 를 누르면 발표자 창이 떠요 — 끝!

강의장에서는:

1. 빔프로젝터 연결 후 디스플레이를 **"확장"** 모드로 (미러링 끄기 — 맥은 `⌘+Fn+F1`로 토글, 윈도우는 `Win+P`)
2. 슬라이드 창을 프로젝터 화면으로 드래그 → **`F`** 로 전체화면
3. 발표자 창은 내 노트북에 — 하단에 **"● 강의장 화면 연결됨"** 초록불 확인

## 주요 기능

### 발표자 창
| 기능 | 설명 |
|---|---|
| 현재 / 다음 슬라이드 | 크기 비율은 드래그 스플리터로 자유 조절 (기억됨) |
| 강사 노트 | 슬라이드별 멘트·큐. `⏰` 시간 큐는 주황, `⚠️` 주의는 빨강, `백업` 줄은 파랑으로 자동 강조 |
| 다음 노트 미리보기 | 노트 칸 아래 다음 슬라이드 노트 첫 줄이 흐리게 |
| 필름스트립 | 전체 슬라이드 썸네일 — 클릭 이동, 노트 있는 장엔 ✎ 배지, 크기 슬라이더 |
| 타이머 | 시작/일시정지 토글 · **목표 시간 설정 → 남은 시간** (5분 전 주황·초과 빨강+알림) · 리셋은 두 번 눌러 확정 · 창을 닫아도 이어짐 |
| 현재 시각 | 시계 따로 볼 필요 없어요 (클릭하면 초 단위) |
| 슬라이드 체류 시간 | 이 장에서 얼마나 오래 말하고 있는지 |
| 연결 상태 | 강의장 화면과의 동기화를 2초마다 자동 확인 (●/○) |
| 라이트 테마 | ☀️ 버튼으로 흰 배경 전환 (밝은 강의장용, 기억됨) |

### 강의장 화면(청중) 배려
- **B / W 블랙아웃** — 강의장 화면만 검게/희게 (쉬는 시간·집중 유도). 슬라이드를 넘기면 자동 해제
- 4초간 조작 없으면 하단 힌트·커서 자동 숨김 (프로젝터에 깨끗한 슬라이드만)
- 발표 중엔 강의장 화면에서 도움말·점프 입력이 떠도 청중에게 노출되지 않게 차단
- Wake Lock — 발표 중 화면 꺼짐 방지

### 진행
- **프레젠터 리모컨(클리커) 호환** — Space·Enter·PgUp/PgDn 전부 인식
- **숫자 + Enter** — 슬라이드 번호로 즉시 점프 (Q&A 때 "아까 그 화면" 소환)
- 마지막 장에서 →를 눌러도 처음으로 안 튀어요 (경계 안내만)
- 어느 창에서 넘겨도 두 창이 자동 동기화, 어긋나면 2초 안에 자동 복구

## 단축키

발표자 창 하단 **[? 단축키]** 버튼(또는 `?`)을 누르면 화면에서 바로 볼 수 있어요.

| 키 | 동작 |
|---|---|
| `←` `→` · `Space` · `Enter` · 리모컨 | 슬라이드 이동 |
| 숫자 + `Enter` | 해당 번호로 점프 |
| `Home` / `End` | 처음 / 마지막 |
| `B` / `W` | 강의장 화면 검게/희게 (한 번 더 또는 슬라이드 넘기면 해제) |
| `F` 또는 `F5` | 전체화면 |
| `P` · `⌘K`(맥) · `Ctrl+K`(윈도우) | 발표자 창 열기 |
| `V` | 발표자 창 보기 전환 (다음 슬라이드 ↔ 필름스트립) |
| `T` / `R` | 타이머 시작·정지 / 리셋(두 번) |
| `?` 또는 `H` | 단축키 도움말 |

## 내 슬라이드에 붙이는 법

`index.html`이 곧 템플릿이에요. 세 군데만 바꾸면 됩니다.

**1. 슬라이드 교체** — `<!-- ▼▼▼ 여기부터 여러분의 슬라이드로 교체하세요 -->` 주석 사이의 `<section class="slide">` 들을 여러분의 슬라이드로 바꿔요. 규칙은 두 가지뿐:

```html
<section class="slide">   <!-- 슬라이드 1장 = section.slide 하나 -->
  <!-- 1600×900 기준으로 디자인 (자동으로 화면에 맞게 스케일돼요) -->
</section>
```

- 첫 슬라이드에만 `class="slide active"`
- 내부는 완전 자유 — 어떤 HTML/CSS든 OK

**2. 노트 채우기** — `<script>` 안의 `NOTES` 배열에 슬라이드 순서대로 멘트를 적어요. 없는 장은 빈 문자열 `` ` ` ``.

```js
const NOTES = [
/*01*/ `오프닝 멘트. ⏰ 10:30 컷라인 — 시간 큐는 주황으로 강조돼요.`,
/*02*/ ``,   // 노트 없는 장
/*03*/ `⚠️ 이 시연은 와이파이 필요. 백업: backup.png`,
];
```

**3. 덱 이름 정하기** — `DECK_ID`를 덱마다 다르게. (같은 브라우저에서 여러 덱을 써도 슬라이드 위치·타이머가 서로 안 섞이게 하는 열쇠예요)

```js
const DECK_ID='my-lecture-2026';
```

> 팁: AI(클로드 등)에게 이 저장소 링크를 주고 "내 슬라이드에 이 발표자 모드 붙여줘"라고 하면 알아서 해줘요.

## 어떻게 동작하나요

- 같은 HTML 파일을 **창 2개**로 여는 구조예요. URL 해시가 `#presenter`면 발표자 UI를, 아니면 슬라이드를 렌더링해요.
- 두 창은 `localStorage` + `BroadcastChannel`로 슬라이드 인덱스를 실시간 동기화해요. 전부 브라우저 안에서만 일어나서 **인터넷이 필요 없어요.**
- 발표자 창이 2초마다 ping을 보내고 슬라이드 창이 pong으로 응답해요 — 이걸로 연결 상태 표시와 어긋남 자동 복구를 해요.
- 타이머·테마·노트 크기·레이아웃은 `localStorage`에 저장돼서 창을 닫았다 열어도 유지돼요.
- 발표자 창의 슬라이드 미리보기·썸네일은 실제 슬라이드 DOM을 `cloneNode`로 복제한 뒤 `transform: scale()`로 축소한 것 — 이미지 캡처가 아니라서 항상 원본과 똑같아요.

## 요구 사항

- **크롬 또는 엣지** 권장 (BroadcastChannel·Wake Lock 완전 지원)
- 맥·윈도우 모두 지원
- 서버·빌드·설치 불필요 — `file://`로 그냥 열면 됩니다

## 자주 묻는 질문

**Q. 발표자 창을 실수로 닫았어요.**
`P`로 다시 열면 현재 슬라이드·타이머까지 그대로 따라와요.

**Q. 팝업이 차단됐대요.**
주소창 오른쪽 팝업 아이콘에서 "항상 허용" 후, 화면 안내의 [허용했어요 — 다시 열기] 버튼을 누르면 돼요.

**Q. PDF로도 뽑고 싶어요.**
크롬 인쇄(`⌘P`)에서 PDF 저장 — 발표자 UI는 인쇄에서 자동 제외돼요.

**Q. 두 대의 컴퓨터에서 쓸 수 있나요?**
아니요 — 한 컴퓨터의 화면 2개(노트북+프로젝터) 구조예요. 동기화가 브라우저 안에서만 일어나기 때문이에요.
